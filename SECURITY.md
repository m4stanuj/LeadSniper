# Security Policy

## Supported Versions

LeadSniper's autonomous engine receives security updates based on the current stable release.

| Version | Supported          |
| ------- | ------------------ |
| 2.1.x   | :white_check_mark: |
| 2.0.x   | :white_check_mark: |
| 1.0.x   | :x:                |

## Reporting a Vulnerability

As this tool touches highly sensitive data streams (SMTP credentials, OAuth Tokens, Intent Scoring LLMs), security is paramount.

If you discover a vulnerability in LeadSniper's extraction pipeline, token rotation, or mesh integration, please **DO NOT** open a public issue.

Instead, initiate encrypted comms:
1. Email: `m4stanuj@gmail.com`
2. Subject: `[SECURITY] Vulnerability Report - LeadSniper`

We aim to acknowledge all reports within 24 hours and patch critical zero-days within 48 hours. Bug bounties are awarded based on severity scoring.
